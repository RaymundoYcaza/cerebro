# Deep Research

## Purpose

Build a structured, well-supported synthesis on a topic using the vault plus external technical references when needed.

## When to use

Use this skill when the user asks to:

- Research a technical topic deeply.
- Compare approaches.
- Design a system or plan.
- Produce a recommendation based on evidence.

## Operating rules

- Start from the vault if the topic exists there.
- Use external sources only when the vault is insufficient.
- Separate facts, interpretation, and recommendation.
- Prefer direct evidence over assumptions.
- State tradeoffs clearly.

## Workflow

1. Define the question precisely.
2. Search the vault for relevant context.
3. Search external references if needed.
4. Compare options.
5. Synthesize into a recommendation.
6. Call out risks, dependencies, and unknowns.

## Output format

- Short answer
- Evidence
- Tradeoffs
- Recommendation
- Next steps

## Example

Question: "What is the best next step for Cerebro?"
Answer:

- Best next step: solidify the systemd watcher and auto-reindex flow.
- Evidence: current qmd setup, existing timer, restore script, vault structure.
- Tradeoff: low effort now, high leverage later.
