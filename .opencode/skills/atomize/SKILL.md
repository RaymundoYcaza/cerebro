# Atomize

## Purpose

Turn a raw idea, inbox note, or conversation into a clean atomic note for the vault.

## When to use

Use this skill when the user asks to:

- Convert an idea into an atom.
- Clean up an inbox note.
- Draft frontmatter and a note structure.
- Split a large concept into one self-contained note.

## Operating rules

- One idea per note.
- Prefer evergreen content.
- Do not mix project logistics with conceptual knowledge.
- Use clear Spanish titles when the vault is in Spanish.
- Preserve original meaning, but rewrite for clarity.
- If the note belongs in `Atlas/Dots/`, keep it concise and durable.

## Workflow

1. Extract the core idea.
2. Decide whether it is an atom, MOC, effort note, or inbox note.
3. Draft title, tags, created date, and type.
4. Write the note body in concise paragraphs.
5. Suggest 2 to 5 links to related notes.
6. If needed, suggest a MOC where it should be linked.

## Output format

- Title
- Frontmatter
- Body
- Related notes
- Suggested placement

## Example

Title: Reindexado automático del vault

## Frontmatter

title: Reindexado automático del vault
tags: [qmd, systemd, vault]
created: 2026-04-05
type: atom

---

Body:
El vault debe reindexarse automáticamente cuando cambian notas...
